# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: Yad2 Scraper + LLM Pipeline.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase Scope Note

Phase 2 is a backend-only phase. Deliverables are:
- `backend/app/scrapers/yad2.py` — Yad2 scraper (httpx-first, Playwright fallback)
- `backend/app/llm/verifier.py` — LLM verification/normalization pipeline
- `backend/tests/test_yad2_scraper.py` and `test_llm_verifier.py` — test suites

No new React components, pages, or routes are built in Phase 2. The existing React scaffold
(`frontend/src/App.jsx`) is untouched — it shows "המערכת בבנייה" (system under construction)
until Phase 4.

The UI-SPEC for this phase therefore governs two surfaces:
1. **CLI / terminal output** — what the developer sees when running `python -m scrapers.yad2`
2. **Frozen placeholder UI** — the React scaffold that must not regress during this phase

The shadcn gate is deferred to Phase 4 (Map Web UI) — the first phase with real component work.

## Typography

All values use Noto Sans Hebrew. Phase 2 does not render any new type — this declares the project-wide type scale that Phase 4 will implement. The frozen placeholder uses Tailwind defaults compatible with these values.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Listing descriptions, paragraph text |
| Label | 14px | 500 | 1.4 | Filter labels, metadata, badges, timestamps |
| Heading | 20px | 700 | 1.2 | Section headings, listing card titles |
| Display | 28px | 700 | 1.15 | Page-level heading (app title, empty-state heading) |

Maximum 2 weights in active use: **400 (regular)** and **700 (bold)**. Weight 500 is reserved for labels only; it does not appear in body paragraphs or headings.

**Hebrew note:** Noto Sans Hebrew at 16px/400/1.5 is the minimum readable size for Hebrew script on mobile screens. Do not go below 14px for any Hebrew body text.

## Phase 2 Interaction Contract

Phase 2 has no interactive UI components. The interaction contract for this phase is the **scraper invocation interface**:

| Interaction | Contract |
|-------------|----------|
| Manual run | `python -m scrapers.yad2` — exits 0 on success, exits 1 on unrecoverable error |
| Output format | Structured log lines to stdout using Python `logging` with `%(levelname)s` prefix |
| Error isolation | Any exception inside `run_yad2_scraper()` is caught; `ScraperResult.errors` is populated; process exits cleanly with code 1; no exception propagates to the caller |
| DB write confirmation | After run: query `SELECT count(*) FROM listings WHERE source='yad2'` — visible count increase confirms data flow |
| LLM confidence visibility | Listings with `llm_confidence < 0.7` are in the DB with the flag set — verify via `SELECT source_id, llm_confidence FROM listings WHERE source='yad2' ORDER BY llm_confidence ASC LIMIT 10` |
