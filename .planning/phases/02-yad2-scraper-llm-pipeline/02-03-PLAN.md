           "confidence": {"type": "number"},
           "price": {"type": ["integer", "null"]},

   }

   **`VERIFY_PROMPT`** — Hebrew-aware prompt for Claude:
   ```python
   VERIFY_PROMPT = """You are analyzing a Hebrew real estate listing from Yad2 (יד2).

   Determine if this is a genuine RENTAL listing. Reject if it is:
   - A "looking for apartment" post (מחפש/ה דירה)
   - A sale listing (למכירה / מכירה)
   - Spam, advertising, or unrelated content
   - A roommate search (שותף/ה)

   If it IS a rental listing, extract these fields:
   - price: Monthly rent in ILS (integer, no currency symbol)
   - rooms: Number of rooms (float, e.g., 2.5, 3, 3.5)
   - size_sqm: Apartment size in square meters (integer)
   - address: Full address in Hebrew (street, neighborhood, city)
   - contact_info: Contact name and/or phone number

   Set confidence (0.0-1.0) based on how clearly the text identifies as a rental and how many fields you could extract. High confidence (>0.8) means clear rental with most fields extractable. Low confidence (<0.5) means ambiguous text or very few fields found.

   Listing text:
   {text}"""
   ```

   **`def get_llm_client() -> anthropic.AsyncAnthropic`**: Returns `anthropic.AsyncAnthropic(api_key=settings.llm_api_key)`. Extracted as a function for easy mocking in tests.

   **`async def verify_listing(raw_text: str) -> dict`**:
   - Calls `get_llm_client().messages.create()` with:
     - `model=settings.llm_model`
     - `max_tokens=512`
     - `messages=[{"role": "user", "content": VERIFY_PROMPT.format(text=raw_text)}]`
     - `output_config={"format": {"type": "json_schema", "schema": LISTING_SCHEMA}}`
   - Parses `json.loads(response.content[0].text)`
   - Returns the parsed dict
   - On exception: logs error, returns `{"is_rental": False, "rejection_reason": f"LLM error: {e}", "confidence": 0.0, "price": None, "rooms": None, "size_sqm": None, "address": None, "contact_info": None}`

   **`async def batch_verify_listings(raw_texts: list[str]) -> list[dict]`**:
   - Per D-03: uses `asyncio.gather(*[verify_listing(t) for t in raw_texts], return_exceptions=True)`
   - Filters results: exceptions become rejection dicts with `"rejection_reason": f"LLM error: {e}"`
   - Returns list of result dicts (same length as input)
   - Logs: `[llm] Verifying {N} listings with {settings.llm_model}`
   - Logs: `[llm] {accepted} accepted, {rejected} rejected, {flagged} flagged`

   **`def merge_llm_fields(scraper_data: dict, llm_data: dict) -> dict`**:
   - Per LLM-04: scraper non-null fields take precedence, LLM fills nulls.
   - Merge fields: `price`, `rooms`, `size_sqm`, `address`, `contact_info`
   - For each field: `merged[field] = scraper_data.get(field) if scraper_data.get(field) is not None else llm_data.get(field)`
   - Also set `merged["llm_confidence"] = llm_data.get("confidence", 0.0)`
   - Returns merged dict
