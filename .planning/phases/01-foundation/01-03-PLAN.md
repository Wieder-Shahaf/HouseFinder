Based on the VPS provider decision from Task 1, the user provisions the server and deploys the app.

**If Option A (Kamatera):**
1. Go to https://www.kamatera.com/ and create an account (30-day free trial)
2. Create a server: Ubuntu 22.04/24.04 LTS, 1 CPU, 1GB RAM, 20GB SSD, Israel region (Petah Tikva or Haifa)
3. Note the server IP address

**If Option B (DigitalOcean):**
1. Go to https://education.github.com/pack and claim the $200 DigitalOcean credit (D-10)
2. Go to https://cloud.digitalocean.com/ and create an account using the Student Pack
3. Create a Droplet: Ubuntu 22.04/24.04 LTS, Basic, $6/mo (1 CPU, 1GB RAM, 25GB SSD), Amsterdam (AMS3) or Frankfurt (FRA1)
4. Note the Droplet IP address

**For both options — after server is created:**
5. SSH into the server: `ssh root@YOUR_SERVER_IP`
6. Install Docker and Docker Compose:
   ```bash
   curl -fsSL https://get.docker.com | sh
   ```
7. Clone the repo:
   ```bash
   git clone https://github.com/YOUR_USERNAME/ApartmentFinder.git
   cd ApartmentFinder
   ```
8. Create `.env` file from `.env.example`:
   ```bash
   cp .env.example .env
   nano .env   # fill in real values later
   ```
9. Update `nginx/nginx.prod.conf`: replace `yourdomain.me` with your actual domain name

**Domain setup (D-11):**
10. Go to https://education.github.com/pack and claim the free Namecheap .me domain
11. In Namecheap dashboard, set an A record pointing to your VPS IP address
12. Wait for DNS propagation (usually 5-30 minutes)

**SSL and deployment:**
13. On the VPS, get the initial SSL certificate:
    ```bash
    # First, temporarily start nginx without SSL (comment out the 443 server block)
    # Then run certbot standalone or use the webroot method
    docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
    docker compose exec certbot certbot certonly --webroot -w /var/www/certbot -d yourdomain.me --email your@email.com --agree-tos --no-eff-email
    ```
14. Restart the stack to pick up the SSL certificate:
    ```bash
    docker compose -f docker-compose.yml -f docker-compose.prod.yml down
    docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d
    ```
15. Verify: open `https://yourdomain.me` in a smartphone browser — you should see the Hebrew placeholder page

  <action>
Complete the Twilio WhatsApp setup per D-14 through D-17. This is a manual task because it requires account creation and UI interaction.

**Steps:**
1. Go to https://www.twilio.com/try-twilio and create a free account
2. Verify your phone number during signup
3. In the Twilio Console, go to **Messaging > Try it out > Send a WhatsApp message**
4. Follow the sandbox setup instructions — send the join code from your WhatsApp to the Twilio sandbox number
5. Note the sandbox credentials:
   - Account SID: found on the Twilio Console dashboard
   - Auth Token: found on the Twilio Console dashboard (click "Show")
   - WhatsApp From number: the sandbox number (format: `whatsapp:+14155238886`)

**Submit message template for Meta approval (D-16):**
6. In the Twilio Console, go to **Messaging > Content Template Builder**
7. Click "Create new" and create a template:
   - Name: `apartment_finder_new_listings`
   - Category: `UTILITY`
   - Language: `he` (Hebrew)
   - Body: `{{1}} דירות חדשות נמצאו בחיפה. פתח את האפליקציה: {{2}}`
   - Variables: `{{1}}` = listing count (integer), `{{2}}` = app URL (string)
   - Note: Twilio Content Template Builder uses `{{1}}`, `{{2}}` placeholders, not `{{count}}`, `{{url}}`
8. Submit for approval

**Save credentials:**
9. On your VPS, update the `.env` file:
   ```
   TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   TWILIO_AUTH_TOKEN=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   TWILIO_WHATSAPP_FROM=whatsapp:+14155238886
   ```
10. Test the sandbox connection (optional):
    ```bash
    curl -X POST https://api.twilio.com/2010-04-01/Accounts/$TWILIO_ACCOUNT_SID/Messages.json \
      --data-urlencode "From=whatsapp:+14155238886" \
      --data-urlencode "To=whatsapp:+YOUR_PHONE" \
      --data-urlencode "Body=ApartmentFinder test message" \
      -u "$TWILIO_ACCOUNT_SID:$TWILIO_AUTH_TOKEN"
    ```
