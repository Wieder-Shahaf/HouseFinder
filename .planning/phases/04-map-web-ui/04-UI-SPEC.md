## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding |
| sm | 8px | Card internal element spacing, button padding |
| md | 16px | Card padding, sheet internal padding, filter row gaps |
| lg | 24px | Bottom sheet top padding, section gaps inside sheet |
| xl | 32px | Bottom nav height clearance (padding-bottom for map container) |
| 2xl | 48px | Filter sheet top padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Bottom navigation bar height: 56px (14 * 4 — matches mobile OS safe-area convention; use `h-14` in Tailwind)
- Touch targets (buttons, toggles, nav tabs): minimum 44px tall — use `min-h-[44px]` where needed (accessibility requirement)
- Leaflet map container: `calc(100dvh - 56px)` — accounts for bottom nav bar; use `100dvh` not `100vh` for mobile browser chrome

**Source:** Standard 8-point scale + mobile touch target standard (REQUIREMENTS.md MAP-08) + CONTEXT.md D-09.

## Interaction Contracts

### Pin States (MAP-05, MAP-06, D-10)
- **Unseen → Seen:** Tap "ראיתי" in `ListingSheet` → `PUT /api/listings/{id}/seen` → optimistic update in TanStack Query cache → pin becomes grey at 50% opacity → sheet dismisses.
- **Unseen → Favorited:** Tap "שמור" in `ListingSheet` → `PUT /api/listings/{id}/favorited` → pin becomes red heart icon.
- **Default map view** shows all pins including seen ones (API default). Seen pins are visually de-emphasized (grey, 50% opacity) but not hidden at map level — consistent with Phase 3 API behavior where `is_seen` filter is opt-in.

### Bottom Sheet (D-01, D-02, D-03)
- Slides up from bottom edge on pin tap.
- Fixed height: `h-[60vh]` — enough to show all fields without scrolling.
- Dismiss: tap anywhere on map (overlay with `onClick` on map container).
- No drag handle interaction — fixed height only.
- Z-index above map tiles but below browser chrome.

### Filter Sheet (D-04, D-05)
- Opens via filter icon button in top-right corner of map view.
- Same sheet component as `ListingSheet` but with filter controls.
- Filters apply live — each change calls `refetch()` via TanStack Query with updated params (D-05).
- API params used: `price_max`, `rooms_min`, `rooms_max`, `neighborhood`, `since_hours` (value: `24` when "חדשות בלבד" is on).
- Room count toggle maps to: 2.5 rooms → `rooms_min=2.5&rooms_max=2.5`; 3 → `rooms_min=3&rooms_max=3`; 3.5+ → `rooms_min=3.5` (no max).
- Multiple room toggles can be active simultaneously — frontend sends one request per active toggle and merges results, or sends the lowest `rooms_min` of active selections.

### TanStack Query Config (D-11)
- `staleTime`: 5 minutes (300,000 ms)
- `refetchOnWindowFocus`: true
- `refetchInterval`: 300,000 ms (5 minutes) while tab is active
- Query key: `['listings', filterParams]` — changes when any filter changes

### Map Viewport (MAP-03)
- Default center: `[32.7940, 34.9896]` (Haifa city center)
- Default zoom: `13` — shows Carmel, Downtown (Merkaz HaIr), and Neve Shanan in one view
- Tile provider: OpenStreetMap (free, no API key) — `https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`

### Routing (D-08, D-09)
- `/` → `MapView` (default landing)
- `/favorites` → `FavoritesView`
- `BrowserRouter` wraps app at root in `App.jsx`
- `BottomNav` rendered in `App.jsx` outside `<Routes>` — persistent on both routes

### index.html Requirements (CONTEXT.md code_context)
- `<html dir="rtl" lang="he">` — mandatory from day one (MAP-07)
- Viewport meta: `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` — `viewport-fit=cover` needed for iOS safe areas with bottom nav

## API Contract Reference

Exact query param names from `backend/app/routers/listings.py`:

| Frontend filter | API param | Type | Notes |
|-----------------|-----------|------|-------|
| Price slider value | `price_max` | int | Max 4500 |
| Room toggle | `rooms_min` + `rooms_max` | float | See room mapping above |
| Neighborhood toggle | `neighborhood` | str | Hebrew text: כרמל / מרכז העיר / נווה שאנן |
| New only (24h) | `since_hours=24` | int | Hardcoded per D-07 |
| Seen filter | `is_seen` | bool | Not in main filter panel — used internally |
| Favorites fetch | `is_favorited=true` | bool | Used on `/favorites` route |

Mutation endpoints:
- `PUT /api/listings/{id}/seen` — mark seen
- `PUT /api/listings/{id}/favorited` — mark favorited

`ListingResponse` fields used in UI: `id`, `title`, `price`, `rooms`, `size_sqm`, `address`, `contact_info`, `post_date`, `source_badge`, `url`, `lat`, `lng`, `is_seen`, `is_favorited`.
