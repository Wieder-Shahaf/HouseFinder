      - Load subscription from `/data/push_subscription.json` using `pathlib.Path`. If file missing: log warning "No push subscription found — skipping notification" and return.
      - Build payload: `{"title": f"{count} דירות חדשות", "body": "לחץ לפתיחת המפה", "url": settings.app_url}`
      - Call `webpush(subscription_info=subscription, data=json.dumps(payload), vapid_private_key=settings.vapid_private_key, vapid_claims={"sub": f"mailto:{settings.vapid_contact_email}"})` inside try/except WebPushException — catch and log, never raise.
      - On success: stamp `UPDATE listings SET notified_at = datetime.now(timezone.utc) WHERE id IN (listing_ids)`.
      - Commit session.
    - `def send_whatsapp(count: int, url: str, price: Optional[int] = None, rooms: Optional[float] = None, neighborhood: Optional[str] = None) -> None`:
      - Stub: log "WhatsApp notifications deferred — Twilio template not yet approved" and return.
      - Do NOT import twilio SDK. Do NOT make any API calls. This is a placeholder for NOTF-01/NOTF-02.
      - Signature includes price, rooms, neighborhood now so future activation (NOTF-02) does not require changing callers.

    IMPORTANT: Create a fresh `vapid_claims` dict on every `webpush()` call — pywebpush mutates the dict in place (see Research pitfall 1).
